팬·아티스트 전용 공간 설계로 참여율과 경험을 개선한 굿덕 리디자인 프로젝트

Goodduck

아티스트와 팬이 직접 소통할 수 있는 팬덤 플랫폼

Summary

Key Responsibilities

  • UXUI 디자인 : 입사 직후 혼자 전면적 리디자인을 진행해서 사용성을 높이고 브랜드 경험을 개선했어요.
  • UX 리서치 : 설문조사, 유저 인터뷰, 사용성 테스트를 진행해서 인사이트를 도출하고 개선 방향을 수립했어요.

Business Goal

팬과 아티스트 관계를 지속·확장시켜,
팬덤 경험의 중심이 되어야 한다.

굿덕은 리디자인 프로젝트로 표면적 UI 문제나 단순한 기능 개선을 넘어, 재방문율·체류시간 등을 상승시켜
장기적으로 팬덤 내 관계 확장을 촉진하고 플랫폼 성장의 기반을 마련할 수 있다고 생각했어요.

Situation

기존 디자인은 팬들이 기대한 ‘우리만의 친밀한 공간’
경험과는 거리가 있었어요.

사용자 인터뷰 조사 결과 유저들은 굿덕을 팬과 아티스트의 친근하고 소소한 공간이라고 여기고 있었어요.
하지만 기존 굿덕 UI는 사용자가 원하지 않는 기능, 불필요한 플로우 등, 불편한 UX로
굿덕의 핵심가치를 제대로 전달하지 못하고 있었어요.

Behavior

아티스트별 메인화면 출시

이제 팬유저들은 내가 입덕한 아티스트만의 특별한 공간을 만날 수 있게 되었고,
인스타그램처럼 외부 노출이 강한 곳이 아닌, 팬들끼리의 가벼운 글이나 댓글을 자유롭게 쓸 수 있게 되었어요.

핵심 가치 도출

우리는 우리 서비스의 핵심가치를 제대로 전달하고 있는가?

우리는 이 프로젝트가 단순 UI 개선이 아니라,
팬–아티스트 관계를 확장시키는 제품의 본질적 목표와 연결되어야 한다고 생각했어요.
그래서 먼저 팬들이 정말로 기대하는 핵심 가치를 정의하는 데 집중했고,그 관점에서 문제를 다시 인식했어요.

왜 핵심 가치를 찾았는가?

1. 단순히 UI 최적화가 아니라 근본적인 문제를 찾길 원했어요.

현재 서비스 전체를 통틀어 근본적인 문제가 무엇인지 파악해야 했어요.

2. 참여율/행동 데이터만으로는 본질을 설명하지 못했기 때문이에요.

수치적으로는 인식해도 “왜 사용자가 이런 행동을 하는가?”를 설명하려면 사용자가 진짜로 원하는 가치를 이해해야 했어요.

여러 리서치를 진행하며 우리는 서비스의 핵심 가치를 도출할 수 있게 되었어요.

- 아티스트,팬 20명의 유저에게 굿덕 사용 경험을 조사하는 인터뷰를 진행했어요
- 아티스트 5명에게 타 플랫폼과 비교하여 굿덕을 쓰는 이유를 중심적으로 인터뷰를 진행했어요
- 서비스 이용 만족도 설문조사를 진행했어요 (1차~3차 총 160명 참가)

사용자 인사이트

사용자들이 실제로 원하는 핵심 경험은 무엇일까요?

앞선 인터뷰, 설문조사 등으로 그 과정에서 다음과 같은 인사이트를 얻을 수 있었어요.
사용자들은 인스타그램 같은 외부 SNS는 공식적이고 다듬어진 외부 노출 공간이라고 생각하는 반면,
굿덕은 팬들과의 사적인 팬카페 같은 편안한 공간이라고 생각하고 있었어요.
그래서 굿덕에서는 부담 없이 소소한 글을 남기고 싶어했어요.

핵심 문제 정의

굿덕은 핵심 가치를 전달하는데 있어서 어떤 문제를 가지고 있었을까?

지금의 굿덕은 핵심 가치(팬-아티스트만의 친밀한 공간)를 제공하는 것에 있어 구체적으로 어떤 문제를 가지고 있었을까요?
우리는 기존 서비스의 데이터와 사용자 피드백, 경쟁사 벤치마킹 등으로 구체적인 문제를 찾으려고 했어요

핵심 문제 1

사용자가 원하지 않는 기능이 메인에 배치되어,
소통 경험을 가로막고 있었어요.

근거 : 정량 데이터와 사용자 피드백
응원글을 업로드한 유저는 전체의 4.79%에 불과했지만, 로그 화면에 항상 응원글이 보이게끔 위치하고 있었어요.
또한 “다른 팬들의 응원글을 메인에서 보는 건 불편하다”라는 유저 피드백이 꽤 많았어요.

핵심 문제 2

다른 팬덤 서비스에서는 쉽게 제공되는 기능이,
굿덕에서는 여러 단계를 거쳐야 가능했어요.

근거 : 타 서비스 벤치마킹과 정량 데이터
홈화면 상단의 아티스트 카드를 누르면 아티스트 피드를 볼 수 있는 공간으로 진입하는 서비스가 많았어요.
그러나 굿덕은 아티스트 카드를 누르면 아티스트 소개를 볼 수 있었어요.
그 결과, 아티스트 피드(로그 화면)를 보고싶어하는 유저가 뒤로가기를 눌러 다시 홈으로 되돌아가고 있었어요.
홈 에서 아티스트 카드를 클릭 후 프로필로 진입한 유저중 약 60%가 홈으로 되돌아온 후,다시 피드(로그 화면)으로 진입했어요.
즉, 유저의 원래 의도는 ‘로그 화면 보기’ 였는데, 프로필이라는 예상치 못한 경로를 거쳐야 한거죠.

가설 설정

문제 요약 : 지금의 굿덕은 팬덤 내 관계 확장을 어렵게 하고있어요. 구체적으로 기능 배치/플로우 문제를 가지고 있었죠
“굿덕의 팬덤 내 소통 경험 확장 한계는, 사용자가 원하는 핵심 가치와 경험을 디자인이 제대로 반영하지 못했기 때문이다.”
이를 해결할 방법으로 어떤 가설이 필요할까요?

솔루션 중심 가설

“우리가 팬들이 원하는 핵심 경험을 홈 화면에서 직접·쉽게 제공한다면,
자연스러운 참여가 증가하고 팬–아티스트 관계는 확장될 것이다.”

원인과 문제 중심의 서브 가설

접근성 부족

사용자가 원하는 콘텐츠에 쉽게 접근하지 못했기 때문에, 자연스러운 참여가 저해되었다.

기능 배치 오류

메인 화면의 정보 구조가 사용자의 기대와 맞지 않았기 때문에, 팬–아티스트 관계 경험이 단절되었다.

디자인 솔루션 탐색

가설을 효과적으로 검증할 최적의 플로우와 레이아웃 찾기

원칙적으로는 A/B 테스트나 사용성 평가를 통해 가설을 체계적으로 검증하는 것이 적절했지만,
프로젝트 특성상 제한된 기간 내 리디자인을 진행해야 했기 때문에, 우리는 정량적 데이터와 경쟁사 벤치마킹을 종합적으로 검토하여 근거를 확보했고,
이를 바탕으로 디자인을 발전시켜 최적의 솔루션을 도출하려고 했어요.
“사용자들이 원하는 팬-아티스트와의 친밀한 공간이라는 핵심가치를 제공하기 위해선 어떤 솔루션이 필요할까?”

경쟁 서비스 벤치마킹

다른 팬덤 서비스들은 아티스트별 독립 공간을 제공하고 있었어요.

→ 아티스트별로 독립적인 공간을 제공해 소속감을 주는 경험을 이끌어내는건 어떨까?

메인 기능 사용률 불일치

메인(로그 화면)에서 응원글 노출이 영역을 크게 차지하는 것에 비해 낮은 사용률과 부정적인 유저 피드백이 있었어요.

→ 메인에 있는 콘텐츠들을 전면적으로 재구성하는건 어떨까?

팬들이 원한 핵심 경험인 “우리만의 친밀한 공간” 이 경험을 가장 직접적이고 명확하게 전달할 방법은
아티스트 전용 공간 메인 화면을 출시하는 것이라고 생각했어요.

이는 단순 UI 개선이 아니라, 팬–아티스트 관계를 확장시키는
핵심 경험을 담은 공간이라는 의미를 담고 있어요.

그렇다면 핵심 가치를 제공하기위한 아티스트 전용 공간은 어떤식으로 구성해야할까?

- 어떻게 홈에서 바로 접근하게 해야할까?
- 아티스트 컬러를 강하게 반영할까 전체적인 Primary 컬러로 통일감을 유지할까?
- 메인 정보 구조는 어떻게 설계해야할까? (어떤 콘텐츠를 우선적으로 배치할까?)

1. 어떻게 홈에서 바로 접근하게 해야할까?

현재 문제에서 말했듯이, 아티스트 카드에서 메인이 아닌 프로필로 진입시키자 60%의 유저가 되돌아가고 있었고,
경쟁 서비스 벤치마킹 결과 아티스트 카드에서 아티스트 메인으로 이동하는 경우가 가장 보편적이고 익숙한 경험이었어요.
그래서 메인화면 상단에 있는 아티스트 카드에서 진입시키기로 결정했어요.

경쟁 서비스 벤치마킹으로 아티스트 탭을 없애는 것도 고려했으나,
이미 많은 유저들이 저곳으로 피드를 보러 이동하는것에 익숙해져 있었고, 기능 개발적으로도 충돌이 있을 수 있어,
프로젝트 기간도 함께 고려하여 유지하는 것으로 결정하고,
여러 아티스트를 팔로우 하고있는 유저에게는 가장 마지막으로 접근한 아티스트 메인을 보여주도록 결정했어요.

2. 아티스트 컬러를 강하게 반영할까 전체적인 Primary 컬러로 통일감을 유지할까?

아티스트만의 독립적인 느낌을 주려면 어떤 시각적 요소들이 필요할까?
아티스트별로 공간이 있다라는 느낌을 주고싶다면, 해당 아티스트의 개성이 강하게 드러나야 했고
팬들이 기대하는 “우리만의 친밀한 공간”이라는 가치가 UI 전반에 녹아 있어야 했어요.
그래서 아티스트 사진을 크게 배치하고, 각 아티스트별 상징 컬러를 부여했어요.
스크롤로 인해 아티스트 사진이 보이지 않아도, 아티스트 컬러가 나타나,
여전히 해당 아티스트의 공간에 있다는 느낌을 줄 수 있어요.

3. 메인 정보 구조는 어떻게 설계해야할까?

어떤 콘텐츠부터 상단에 배치해야 할까?
아티스트 메인은 해당 아티스트의 콘텐츠를 요약해서 보여주는 것과 같아요.
가장 중요한 콘텐츠부터 배치해야해요. 그래서 팬들에게 중요한 공지를 첫번째로 배치했어요. 그러나 공지는 자주 보거나 팬들이 즐기는 콘텐츠는 아니기 때문에
최신 하나만 보이도록 했어요. 두번째로 굿톡 다시듣기를 배치했어요.
굿톡은 보이스 전용 라이브로, 다시듣기는 멤버십 회원에게만 제공되요.
비즈니스로 연결되는 가장 중요한 기능 중 하나기 때문에 두번째로 배치했어요.
팬들이 올린 자유게시판과 응원글은, 팬 유저의 관심도가 가장 낮기 때문에 하단에 배치했어요. (아티스트 홈화면에선 팬 전용 콘텐츠가 가장 상단에 보여요.)

사용성 테스트

리디자인 후, 의도한 흐름대로 사용자가 서비스를 사용하고 있는지,
사용성 이슈가 있는지 검증하기 위해 사용성 테스트를 진행했어요.

사용성 테스트 진행 후 사용자 참여율에 핵심적인 영향이 있는 사용성 문제를 발견할 수 있었어요.

1. 사용자들은 댓글 (미트볼) 버튼이 눈에 띄지 않는다고 주장했어요.

2. 사용자들은 글쓰기를 위한 플러스(+) 버튼이 무엇을 의미하는지 이해하지 못했어요.

개발 공수를 고려하여 작지만 큰 효과를 낼 수 있도록 아래와 같이 개선했어요.

그 외 발견한 오류를 빠르게 즉시 개선했어요.

성과

1. 28.8%의 전체 이탈율 개선

3월에는 홈에서 아티스트 피드로 넘어가지 않고 이탈하는 사용자가 100명 중 약 62명이었는데,
7월에는 그 수가 44명으로 줄었어요. (리디자인은 4월 출시)
즉, 이전에 이탈하던 사람 100명 중 약 29명이 이탈하지 않고 피드로 진입하게 된것이에요!

2. 글쓰기 버튼 개선 후 업로드 수 증가

사용성 테스트가 끝나고 글쓰기 버튼에 텍스트 추가 후
이전 기간 (8월 1일 ~ 13일)과 출시 후 기간인 19일 이후 데이터를 비교해보면,
자유게시판은 이전 대비 약 50% 증가, 응원글은 약 73% 증가했어요.
자유게시판은 사용자 수가 32명에서 48명으로 증가했고, 응원글은 41명에서 무려 71명으로 증가했어요.

3. 실제 유저 반응

회고

단순히 예쁘게가 아니라, 좀 더 편하게 접근 가능한, 확연한 아이덴티티를 가진 굿덕의 핵심가치를
어떻게 UIUX로 표현할 것인가에 많이 고민하는 과정이었어요. 문제를 정의하고, 가설을 세우고,
디자인 솔루션을 만들고, 사용성 테스트로 검증한 뒤 실제 지표 변화를 확인하기까지,
디자인 프로세스의 처음부터 끝까지 경험할 수 있었던 값진 프로젝트였다고 생각해요.
하지만 혼자 UXUI와 리서치를 모두 담당하다 보니 A/B 테스트 같은 실험 설계를 못 했던 점이 아쉬웠어요.
이 경험을 통해, 앞으로는 좀 더 체계적인 리서치 설계와 데이터 기반 실험으로 결과를 뒷받침하고,
초기 단계부터 더 완성도 있는 UXUI 개선을 해보고 싶어요.
